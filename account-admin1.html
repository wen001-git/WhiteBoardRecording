静态账号管理

分别维护账号哈希 accounts.json 和价格/微信 paywall.json；不连接 Neon / Node 后端。

重要：accounts.json 只保存哈希；「明文账号清单」才是发给用户的用户名和密码。 如果你的 GitHub 仓库是公开的，不要把明文账号清单提交到仓库。accounts.json 绝不保存价格或微信号，购买配置只写入独立的 paywall.json。

修改输入会立即同步当前浏览器中的白板预览，但不会改动 accounts.json。点击保存后，Chrome/Edge 可选择并覆盖项目根目录的 paywall.json；其他浏览器会下载文件。

部署到服务器（请逐步完成）：
  1. 填写价格和微信号，点击「保存到 paywall.json」。
  2. 如果浏览器下载了文件，把它覆盖到仓库根目录现有的 paywall.json；不要覆盖 accounts.json。
  3. 在仓库终端执行：git add paywall.json && git commit -m "更新价格和微信" && git push。
  4. 等待 Render 自动部署约 30 秒，再刷新线上白板检查新价格和微信号。
账号发放：生成或合并账号后，另行下载并部署 accounts.json；发给客户的是「用户名 + 明文密码」，不是哈希值。价格配置和账号配置是两套完全独立的文件。

一键生成 Pro 账号

本页静态账号统一盐值：wb-static-pro-salt-v1
该值实时读取自当前 accounts.json 的顶层 salt 字段。「生成账号」只先生成明文；点击“合并并输出”时，生成账号和手动合并都会用这个盐值计算哈希。静态账号与新建/改密后的 Node/Neon 后端账号统一使用此盐值；两边哈希算法不同，但盐值相同。

用户名会生成如 wbpro001。密码至少 4 位，满足当前登录页限制。
